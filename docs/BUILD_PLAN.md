# StorySitting App Build Plan

## We stop spinning by changing the output

No more public-site redesign as the main workstream.

The workstream is now the app engine:

1. Private Story Room
2. Contributor invite
3. Contribution submission
4. Staff review
5. Memory Card
6. Story Map
7. Story Capsule delivery

## Sprint 0.1 acceptance criteria

- Supabase project exists.
- Schema is installed.
- Buckets exist.
- RLS is enabled.
- Owner signup works.
- Owner can create a Story Room.
- Owner can generate invite link.
- Contributor can submit memory and optional file.
- Staff can view contribution queue.
- Staff can approve/archive/create Memory Card.
- Staff can create Story Map.
- Manual Quo import creates ImportEvent + Contribution.

## What is intentionally not included yet

- Stripe checkout
- AI drafting
- full partner portal
- automated phone call-in
- automated Quo webhook
- PDF generator
- white-label client portals
- public sharing
