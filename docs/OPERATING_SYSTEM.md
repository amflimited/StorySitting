# StorySitting Operating System

This document is the operating doctrine for StorySitting. It exists so future development does not restart from scratch every time the founder and GPT shift into a new work session.

## Current Company Mission

StorySitting helps families gather scattered memories, photos, voices, questions, places, objects, recipes, and interview material into finished private Story Capsules.

## Current Wedge

The first sellable product is the Signature Story Capsule.

The offer should not be over-narrowed around farms, barns, old trucks, or any single rural/homeplace image. Those can be valid subcases, but they are not the main category.

The stronger product frame is:

> Story Capsules for people, recipes, traditions, family homes, milestones, anniversaries, remembrance, retirement, reunions, and family chapters worth finishing.

The system should help a family start with the thing they do not want to lose, then guide that material into a Story Map and a finished Capsule.

## Primary Capsule Categories

1. Parent or Grandparent Story Capsule
2. Recipe & Tradition Capsule
3. Family Home Capsule
4. Milestone Birthday Capsule
5. Anniversary Capsule
6. Remembrance Capsule
7. Retirement or Life's Work Capsule
8. Family Reunion Capsule
9. Holiday Tradition Capsule
10. Family Business or Chapter Capsule
11. Custom Story Capsule

## Product Lock

StorySitting is not primarily a SaaS product, photo product, AI transcription tool, memoir agency, or generic archive.

StorySitting is a guided production system for finishing family Story Capsules.

## Core Pipeline

1. Open Story Room
2. Gather family material
3. Review and organize
4. Create Memory Cards
5. Build Story Map
6. Guide session
7. Produce Capsule
8. Deliver Capsule

Every feature must advance one of these stages.

## The Critical Bridge

The Story Map is not the final product. It is the production blueprint.

The missing bridge is:

Story Map -> Capsule Builder -> Capsule Preview -> Final Deliverable

The Capsule Builder should pull the latest Story Map and selected Memory Cards into a draft deliverable structure with sections, quotes, people, places, themes, open questions, included assets, and production next steps.

## Founder Role

The founder owns:

- vision
- taste
- final product direction
- market sensing
- offer prioritization
- real customer judgment
- revenue pressure

The founder should not be forced into constant technical decision-making, file replacement, or manual production chaos.

## GPT / Mission Control Role

GPT owns:

- system architecture
- implementation sequencing
- product workflow design
- UI copy clarity
- production logic
- launch cadence
- reusable component strategy
- debugging and stabilization

GPT should not keep branching strategy mid-build or turning every change into a new website concept.

## Development Modes

### Strategy Mode

Used to decide what matters. No code changes.

### Build Mode

Used to implement one coherent improvement batch. No strategy expansion mid-flight.

### Repair Mode

Used when build or user flow is broken. No feature work until the break is repaired.

### Design Mode

Used only when the product flow is stable and assets are ready.

## Mission Packet Format

Every build mission should define:

- Objective
- Files affected
- Expected result
- Test path
- Stop condition

Example:

Objective: Improve Story Room guidance.
Files affected: Story Room page, workflow component, CSS.
Expected result: owner knows the current phase and next action.
Test path: /story-rooms/[id]
Stop condition: page clearly answers where am I, what next, and what done looks like.

## SpaceX-Style Launch Cadence

1. Define mission
2. Freeze scope
3. Build coherent patch
4. Verify compile/deploy
5. Test one path
6. Capture telemetry/friction
7. Start next mission

No scattered micro-edits unless repairing a specific failure.

## Current Strategic Priority

The next major proof target is one complete canonical Signature Story Capsule sample. The strongest first sample should be broadly relatable, such as a parent/grandparent story, a recipe and tradition story, a milestone birthday, or an anniversary.

The sample becomes sales proof, partner proof, workflow proof, and the model for production quality.

## Decision Filter

Before adding anything, ask:

1. Does this help a family finish a Capsule?
2. Does this reduce founder/operator confusion?
3. Does this move a project to the next production phase?
4. Does this clarify the Signature Story Capsule offer?
5. Does this create reusable leverage for future builds?

If not, it waits.
